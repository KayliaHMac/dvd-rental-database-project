# DVD Rental Database Project

## Overview
This project simulates a DVD rental system using PostgreSQL. The database was designed using an ER diagram and supports rental tracking, late fee calculations, and revenue reporting.

## Key Features
- Relational database design using an ER diagram
- User-defined SQL functions to calculate:
  - Late return status
  - Number of days overdue
- Summary tables to track monthly late fee revenue
- Business-focused SQL queries for reporting and insights

## Tools Used
- PostgreSQL
- SQL
- ER Modeling

## Files Included
- SQL scripts for tables, functions, and queries
- ER diagram illustrating database relationships

This project demonstrates end-to-end data analytics skills including data modeling, business rule implementation, and analytical reporting.
